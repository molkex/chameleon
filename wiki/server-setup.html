Xray MUST use network_mode: host. Bridge NAT causes packet flooding and connection failures for VPN traffic.

Xray config requirements

{
  "dns": {
    "queryStrategy": "UseIPv4",    // DNS resolution IPv4 only
    "servers": ["1.1.1.1", "8.8.8.8"]
  },
  "outbounds": [
    {
      "protocol": "freedom",
      "tag": "DIRECT",
      "settings": {
        "domainStrategy": "UseIPv4"  // Outbound connections IPv4 only
      }
    }
  ]
}
Both dns.queryStrategy AND outbounds.DIRECT.settings.domainStrategy must be UseIPv4. The first controls DNS, the second controls outbound connections after sniffing re-resolves domains.

Diagnostic Commands

Quick health check

# Check all services running
sudo docker ps --format 'table {{.Names}}\t{{.Status}}'

# Check Xray network mode (must be "host")
sudo docker inspect xray --format='NetworkMode: {{.HostConfig.NetworkMode}}'

# Check Xray accepts connections
sudo docker logs xray --since 2m 2>&1 | grep -v api | tail -10

# Test outbound connectivity
curl -4 -sk --max-time 5 -w "HTTP:%{http_code} TIME:%{time_total}s\n" \
  -o /dev/null https://www.gstatic.com/generate_204

# Test IPv6 (should work but we force IPv4)
curl -6 -sk --max-time 5 https://ifconfig.me

Debug Xray connections

# Enable debug logging (TEMPORARY!)
sudo python3 -c "
import json
path='/var/lib/docker/volumes/chameleon-xray-config/_data/config.json'
with open(path) as f: c = json.load(f)
c['log'] = {'loglevel': 'debug'}
with open(path, 'w') as f: json.dump(c, f, indent=2)
"
sudo docker restart xray

# Watch debug logs
sudo docker logs -f xray 2>&1 | grep -v api

# IMPORTANT: revert to warning level after debugging!
# Change 'debug' back to 'warning' and restart

Check for new server deployment

# After deploying a new VPN node, verify:
# 1. Xray uses host networking
sudo docker inspect xray --format='NetworkMode: {{.HostConfig.NetworkMode}}'

# 2. DIRECT outbound has UseIPv4
sudo docker exec xray cat /etc/xray/config.json | \
  python3 -c "import json,sys; c=json.load(sys.stdin); \
  [print(json.dumps(o,indent=2)) for o in c['outbounds'] if o.get('tag')=='DIRECT']"

# 3. IPv6 test
curl -6 -sk --max-time 5 https://www.gstatic.com/generate_204 -w "%{http_code}\n" -o /dev/null
# If this fails, IPv6 is broken — UseIPv4 is mandatory
