UX Flows

Флоу ориентированы на оперативное управление сетью и incident handling. Пользователь должен видеть причину проблемы и следующий шаг без лишних переходов.

Flow 01: Connect with Health Visibility

goal: control + trust
1. Home shows live baseline metrics before connection.
2. Connect launches 3-step tunnel sequence with progress.
3. Connected view exposes health badge and session data.
KPI: connect success > 97%, abandon on connect < 5%.

Flow 02: Manual Route Pinning

goal: predictable latency for power users
1. Open Locations tab and compare RTT per node.
2. Open node details and enable Pin route.
3. Define fallback route for automatic failover.
KPI: successful pin application > 95%.

Flow 03: Active Diagnostics

goal: incident triage in-app
1. Enter Diagnostics tab.
2. Run active test for DNS/tunnel/loss checks.
3. Export JSON report or send directly to support.
KPI: support ticket quality score improves via structured report.

Flow 04: Incident Recovery

goal: recover from endpoint/API failure
1. Incident banner appears with short root cause text.
2. Primary action retries tunnel.
3. Secondary action switches fallback route.
4. If failed, open report and contact support.
KPI: self-recovery in-app > 65%.

Flow 05: Expired Plan Recovery

goal: restore protection with minimal friction
1. Expired state highlights protection pause.
2. Paywall with plan details and restore action.
3. Return to previous route and reconnect automatically.
KPI: renewal completion from expired state > 30%.
