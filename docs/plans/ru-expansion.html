Цель: больше RU-клиентов · Ограничение: один Apple-аккаунт · Принцип: основной MadFrog (live) держим вне риска · ред. 2026-06-04

⛔
Жёсткое правило №1 — не churn'им iOS на общем аккаунте

РКН-снятие аккаунт не банит. Но пачка похожих VPN-приложений = Guideline 4.3 (spam) → Apple терминирует весь аккаунт, включая live-MadFrog с выручкой. Матожидание плохое: +пара сотен скачиваний vs −весь бизнес. Поэтому конвейер burner'ов — только вне Apple-аккаунта (Android/RuStore) или вне сторов (Telegram).

План по фазам

Фаза 1 · неделя 1
Быстрые безопасные победы. Трек A (+1 iOS-витрина) + Трек C (усилить Telegram/сайт). Параллельно. Минимум риска, максимум скорости — данные о конверсии собираем сразу.
Фаза 2 · недели 2–4
Масштаб без риска для Apple. Трек B — Android-клиент + RuStore-аккаунт на ИП. Это даёт настоящий «burner»-конвейер, которого нельзя на iOS. Backend/протокол уже готовы — работа в Android-шелле.

Оптимизация воронки + повтор. A/B онбординга, таймер-офферы, ретеншн. Android-пачки на RuStore по накатанному. iOS-витрину переподавать только с изменениями.
