Платформы — где мы есть

Каждая платформа = отдельный канал к пользователю, но один общий backend и подписка.

iOS live
Основной канал привлечения. Native SwiftUI + NetworkExtension.
macOS live
Отдельный листинг в App Store — десктоп-аудитория.
iOS-виджет live
Удержание на экране: таймер «Защищено», быстрый статус.
Telegram-бот live
Прямая продажа @MadFrogRobot — канал без сторов.
Web / лендинг live
Витрина и оплата: madfrog.online + страницы для App Store.
Админ-панель live
Управление без билда: юзеры, серверы, поддержка, рассылки.
Backend live
Ядро системы: Go + Postgres + Redis на ноде NL.
Android скоро
Канал масштаба (RuStore) — в роадмапе, backend готов.

VPN-ядро

VLESS Reality TCP · sing-box / Xray
→обход DPI и блокировок РКН, маскировка под обычный TLS
2 страны-выхода: NL + Франция
→выбор страны + резерв, если один выход упадёт
RU-релеи: MSK (цепочки) + SPB (bypass)
→обход whitelist-фильтров и throttling у операторов РФ
Hysteria2 / TUIC — готовы, выключены
→запас протоколов, если Reality начнут давить

Аккаунты и вход

Apple · Google · e-mail · анон-триал
→низкий порог входа (триал сразу) + восстановление по Apple ID
Keychain-identity, без анон-демоутов
→платящий не «слетает» на триал — меньше потерь и жалоб
Триал один раз (SEC-01)
→защита от бесконечных бесплатных триалов

Платежи

Apple IAP — 4 подписки (StoreKit 2)
→монетизация западной аудитории через App Store
FreeKassa / СБП / карты
→оплата в РФ без Apple — независимый денежный поток
Возврат → отзыв доступа (SEC-04)
→защита от абуза рефандами, честный учёт срока

Поддержка и удержание

In-app чат (SSE live, вложения, диагностика+лог)
→прямая связь и быстрая помощь — меньше отказов и возвратов
Админ-инбокс (ответы, close/reopen, шаблоны)
→агент-сторона: оператор отвечает из одной панели
APNs push — ответ поддержки
→возвращает пользователя в приложение
Push-рассылка всем
→анонсы и промо на всю базу — без обновления приложения
In-app анонсы + таргетинг (триал/платные/платформа)
→апселл и ретеншн нужному сегменту — без билда

Аналитика и управление

Дашборд: Online / Active / DAU / воронка / события
→видимость — где теряем пользователей и сколько платят
Учёт трафика по всем выходам
→точные метрики использования по каждому юзеру
Регион App Store (storefront)
→видно, из какой страны скачали — для гео-решений
Серверы / протоколы / Shield в админке
→добавить страну/сервер без выпуска новой версии

Инфраструктура

Go + Postgres + Redis + Docker/Nginx (NL)
→единое ядро для всех платформ и каналов
Federated user-sync на выходы
→пользователь работает на любой ноде без задержки
Бэкапы в Backblaze B2 (offsite)
→сохранность данных при сбое ноды
Главный рычаг

Один backend питает всё. VPN-протокол, аккаунты, подписки, поддержка, аналитика — общие. Поэтому новая платформа (Android), новый канал (RuStore, Telegram) или новая фича (анонсы, push) переиспользуют готовое ядро и стоят дёшево. Это и есть основа плана роста.
