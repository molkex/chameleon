Server Selection Flow

  1. User taps server in ServerListView
  2. selectServer(groupTag, serverTag) called
  3. Saves selectedServerTag to ConfigStore (nil for Auto)
  4. Updates UI immediately (checkmark moves)
  5. If VPN connected and server changed:
    1. Builds new config with buildConfigWithSelector(tag) — modifies selector default
    2. Saves to UserDefaults for On-Demand reconnects
    3. Disconnects: commandClient.disconnect(), disableOnDemand(), vpnManager.disconnect()
    4. Polls vpnManager.isConnected every 200ms (max 6s) until disconnected
    5. Reconnects with new config: vpnManager.connect(configJSON: updatedConfig)

Debug Tips

Claude Report Button

In Debug Logs (ladybug icon), tap the copy icon (top right). This creates a compact report with:

  • VPN status and uptime
  • Key events (tunnel start, config source, sing-box start)
  • Errors and warnings
  • Last 30 meaningful log lines (filtered from noise)

Paste directly to Claude for analysis.

Common Log Patterns

# Good: traffic flowing through selected server
outbound/vless[Germany]: outbound connection to 142.251.20.156:443

# Bad: server timeout
outbound/urltest[Auto]: outbound Germany unavailable: context deadline exceeded

# Bad: connection failure
ERROR connection: open connection to X using outbound/vless[Germany]: EOF

# Bad: packet flooding (Docker NAT or network issue)
router: dropped due to flooding
