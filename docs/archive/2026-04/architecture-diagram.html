AFTER (FIXED)

1
App wants to open google.com

4

Route rules: sniff → dns→hijack-dns → ip_is_private → direct

ROOT CAUSE: DNS queries to 172.19.0.2 (TUN interface) were caught by ip_is_private rule and sent to direct outbound where they died. VPN showed "Connected" but no sites loaded.

FIX (2026-04-06): Added route rule {"protocol": "dns", "action": "hijack-dns"} BEFORE ip_is_private. The hijack-dns action (sing-box 1.11+) intercepts DNS and routes it to the built-in DNS module. No dns outbound needed — it was removed in sing-box 1.13. DNS now goes to FakeIP + remote resolution through Proxy.

8. Troubleshooting History (2026-04-06 → 07)

Complete path from "VPN connects but no sites" to working. Reference for future debugging.

Problem 1: DNS goes nowhere

!
Symptom: VPN Connected, zero sites load
Cause: DNS to 172.19.0.2 matched ip_is_private → direct → died
Fix: {"protocol":"dns","action":"hijack-dns"} BEFORE ip_is_private
Gotcha: hijack-dns (HYPHEN), not hijack_dns (underscore)
Commit: 688fd52

Problem 2: dns outbound removed in sing-box 1.13

!
Symptom: App crashes — unknown outbound type "dns"
Cause: sing-box 1.13 removed dns outbound. Both validation AND runtime reject it.
Fix: Use hijack-dns action instead. LibboxCheckConfig also disabled (too strict).

Problem 3: IPv6 + QUIC timeout through relay

!
Symptom: DNS works, but QUIC/IPv6 connections timeout
Cause: SPB relay (nginx stream TCP) can't forward IPv6 or UDP/QUIC
Fix: 1) Block QUIC: udp:443→reject 2) Remove inet6 from TUN+FakeIP 3) iOS no IPv6 fallback
Commit: 9a532b3

Problem 4: Xray 1.8.24 crashes on xhttp

!
Symptom: Xray restart loop — unknown transport protocol: xhttp
Cause: teddysun/xray:1.8.24 too old, no xhttp
Fix: Upgrade to teddysun/xray:25.12.8. NOT v26 — see Problem 5.

Problem 5: Xray v26 Reality INCOMPATIBLE with sing-box 1.13

!
Symptom: ALL connections: reality verification failed
Cause: Xray v26.3.27 changed Reality ServerHello. sing-box 1.13.5 client rejects it. Server-side shows connections accepted OK.
Proof: Xray debug logs: accepted tcp ... [VLESS TCP REALITY >> DIRECT] — server OK, client rejects
Fix: Downgrade to teddysun/xray:25.12.8. Works immediately.
Commit: 6038e66
DO NOT UPGRADE to Xray v26 until sing-box/libbox supports it!

Problem 6: DB lost after docker compose project name mismatch

!
Symptom: {"error":"User not found"} saved as VPN config
Cause: docker compose -p chameleon down -v deleted original DB volume
Fix: App auto-registers new user. Added validation for error responses in cached config.
Prevention: Always use same project name. Don't use -v flag with down.
