0290013 2026-04-08
Fix: server selector reconnect + DE server networking
fix iOS server infra
Problem

When user selects "Germany" directly in the iOS app, pages don't load. Through "Russia → DE" relay — everything works fine. Three separate issues found:

Root Cause 1: Xray IPv6 on DE server

Xray's freedom outbound had no domainStrategy, so it used whatever the OS resolved — including IPv6. OVH VPS has partially broken IPv6: curl -6 https://1.1.1.1 fails, IPv6 to Google is 6x slower than IPv4.

With sniffing enabled (destOverride: ["http","tls"]), Xray re-resolves domains from SNI and may get AAAA records → tries IPv6 → connection fails with EOF on client side.

Xray dns.queryStrategy: UseIPv4 only affects DNS resolution, NOT the freedom outbound. You need BOTH.

Fix:

// In Xray config.json, DIRECT outbound:
{
  "protocol": "freedom",
  "tag": "DIRECT",
  "settings": {
    "domainStrategy": "UseIPv4"
  }
}
Root Cause 2: Xray in Docker bridge network

Xray was running in Docker's default bridge network (172.18.0.3) with port mapping 2096:2096. Every VPN packet went through Docker's userspace NAT proxy + conntrack. This caused:

  • Extra latency on every packet
  • Conntrack table pressure under many concurrent connections
  • sing-box on iPhone reported dropped due to flooding and EOF
VPN/proxy servers should ALWAYS use network_mode: host in Docker. Bridge NAT is for web apps, not for high-throughput packet forwarding.

Fix in docker-compose.yml:

# BEFORE (broken):
xray:
  ports:
    - "2096:2096"
    - "2097:2097"
    - "2098:2098"
  cap_drop:
    - ALL
  cap_add:
    - NET_BIND_SERVICE

# AFTER (fixed):
xray:
  network_mode: host
Root Cause 3: iOS Auto button didn't reconnect

When switching from Germany → Auto, the "Auto (best ping)" button only set selectedServerTag = nil without triggering VPN reconnect. User stayed on Germany until manual disconnect/connect.

Fix in MainView.swift:

// BEFORE:
Button {
    app.configStore.selectedServerTag = nil
    dismiss()
}

// AFTER:
Button {
    app.selectServer(groupTag: "Proxy", serverTag: "Auto")
    dismiss()
}

Lessons Learned

Docker networking for VPN/proxy

Always use network_mode: host for Xray/sing-box/proxy containers. Bridge mode adds NAT overhead that kills VPN throughput and causes connection flooding.

IPv6 on VPS

Always test IPv6 outbound on new servers. OVH VPS can have broken IPv6. Add domainStrategy: UseIPv4 to both Xray DNS config AND freedom outbound.

sing-box selector vs urltest

If urltest marks a server as "unavailable", the selector may not respect its default setting. Always ensure the selected server passes health checks, or DNS will work (via Auto) but pages won't load (via broken selector target).

iOS VPN reconnect

When switching servers on a connected VPN: must disconnect, wait for actual disconnect (poll status), then reconnect. Using Task.sleep with fixed delay is unreliable.
