Issue 2 — Guideline 2.3.12 ("What's New" text) on a first-version submission
-----------------------------------------------------------------------------

App Review has rejected this submission with Guideline 2.3.12 in round 2 (resolved at the time after App Review Notes explanation), round 5, and now round 6. The flag is on the string "Initial release" appearing in the "What's New" field.

The problem:

This is the app's first ever release. Because there is no prior published version, App Store Connect:

  1. Does NOT show a "What's New" field anywhere in the version page for a first-version submission (the field does not exist in the ASC web UI).

  2. Returns HTTP 409 STATE_ERROR from the App Store Connect REST API on every attempt to set whatsNew:

       PATCH /v1/appStoreVersionLocalizations/{id}
         Body: {"data":{"type":"appStoreVersionLocalizations","id":"...","attributes":{"whatsNew":"..."}}}
         Response: 409
         {
           "errors": [{
             "id": "...",
             "status": "409",
             "code": "STATE_ERROR",
             "title": "The request cannot be fulfilled because of the state of another resource.",
             "detail": "Attribute 'whatsNew' cannot be edited at this time"
           }]
         }

     This 409 is reproducible in every appStoreState we have tried (REJECTED, PREPARE_FOR_SUBMISSION), with and without an active reviewSubmission, with and without an attached build.

  3. The literal string "Initial release" shown as the value of whatsNew is the App Store Connect default placeholder for first releases — not something we wrote.

We have placed an explicit explanation of this in the App Review Notes, but the 2.3.12 finding has been raised repeatedly anyway.

Questions:

  1. Is "What's New" supposed to be uneditable for a first-version App Store submission? If yes, where is the first-version equivalent of release notes meant to be entered, so that 2.3.12 has something other than the default "Initial release" placeholder to evaluate?

  2. If the field should be editable, what state transition or precondition unlocks it? We have tried submitting / unsubmitting, attaching / detaching builds, REJECTED → PREPARE_FOR_SUBMISSION transitions — all return the same 409.

  3. If neither — can App Review please be informed that "Initial release" on a first-version submission is the App Store Connect default placeholder, and that 2.3.12 should not be raised against it?
