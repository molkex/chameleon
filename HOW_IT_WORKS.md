# Как работает Chameleon VPN — просто о сложном

## Что вообще делает VPN?

Представь, что интернет — это город с дорогами. Когда ты открываешь сайт, твоя машинка (iPhone) едет по дороге к магазину (сайту). Все видят, куда ты едешь — и полиция (провайдер), и соседи.

VPN — это **секретный тоннель**. Машинка заезжает в тоннель у дома, а выезжает уже в другом городе (в Германии или Нидерландах). Никто не видит, куда ты на самом деле едешь.

---

## Наши "тоннели" (серверы)

У нас 3 сервера:

```
🇷🇺 SPB (Россия, 185.218.0.43)     — "дверь в тоннель" рядом с домом
🇩🇪 DE  (Германия, 162.19.242.30)   — "выход из тоннеля" в Европе
🇳🇱 NL  (Нидерланды, 194.135.38.90) — ещё один "выход" в Европе
```

### Как машинка едет?

**Вариант 1 — Через relay (SPB):**
```
iPhone → SPB (Россия) → DE (Германия) → Интернет
iPhone → SPB (Россия) → NL (Нидерланды) → Интернет
```
Это как сначала зайти в подземный переход у дома, а потом по нему доехать до Европы. Работает даже когда прямая дорога в Европу перекрыта.

**Вариант 2 — Напрямую:**
```
iPhone → DE (Германия) → Интернет
iPhone → NL (Нидерланды) → Интернет
```
Быстрее, но если РКН заблокировал IP сервера — не работает.

---

## Протоколы — "виды тоннелей"

Тоннели бывают разные. У нас два вида:

### VLESS Reality (основной)
Представь, что тоннель **притворяется обычной дорогой**. Полиция (DPI) смотрит на него и думает: "а, это просто кто-то заходит на сайт ads.x5.ru" (это реальный русский сайт). А на самом деле внутри — VPN трафик.

- Работает через TCP (надёжно, как почта — каждое письмо доставлено)
- Проходит через relay серверы (SPB → DE, SPB → NL)
- Главный протокол, работает почти всегда

### Hysteria2 (быстрый)
Представь **скоростной поезд** вместо обычной машины. Летит быстрее, но по особым рельсам (UDP/QUIC).

- Работает через UDP (быстро, как разговор — не ждёт подтверждения)
- НЕ проходит через relay (SPB умеет пересылать только TCP, а это UDP)
- Работает только напрямую: iPhone → DE или iPhone → NL
- Если провайдер блокирует UDP порт 8443 — не работает

## Что видит пользователь в приложении

6 серверов в списке:

```
🇷🇺 Russia → DE          — VLESS через SPB в Германию
🇷🇺 Russia → NL          — VLESS через SPB в Нидерланды
🇩🇪 Germany              — VLESS напрямую в Германию
🇳🇱 Netherlands           — VLESS напрямую в Нидерланды
🚀 Germany Hysteria2     — Быстрый UDP в Германию
🚀 Netherlands Hysteria2 — Быстрый UDP в Нидерланды
```

**Auto (best ping)** — приложение само проверяет кто быстрее и выбирает лучший.

## DNS — "телефонная книга интернета"

Когда ты набираешь "google.com", iPhone не знает адрес. Ему нужно позвонить в справочную (DNS) и спросить: "какой IP у google.com?"

### Проблема
Если справочная далеко или заблокирована — ждёшь 30 секунд пока ответят. Сайты грузятся медленно.

### Наше решение — FakeIP
Вместо того чтобы звонить в справочную, мы **выдумываем адрес**:
```
google.com → 198.18.0.1 (фейковый)
youtube.com → 198.18.0.2 (фейковый)
```

iPhone сразу "едет" на фейковый адрес. А sing-box (наш движок) видит: "ага, кто-то едет на 198.18.0.1, а я знаю что это google.com" — и отправляет запрос через VPN на настоящий Google.

**Результат**: сайты открываются мгновенно, нет ожидания DNS.

## Как работает приложение — пошагово

### Первый запуск (после установки)

```
1. Открыл приложение
2. Приложение: "Я новый! Надо зарегистрироваться"
3. → Отправляет device_id на сервер (POST /api/mobile/auth/register)
4. ← Получает: username, access_token, refresh_token
5. → Запрашивает конфиг (GET /api/v1/mobile/config?username=...)
6. ← Получает JSON конфиг для sing-box (все 6 серверов, DNS, маршруты)
7. Сохраняет конфиг в App Group (общая папка для приложения и VPN расширения)
8. Показывает кнопку подключения
```

### Нажатие кнопки "Подключить"

```
1. Есть кеш конфига? → Подключаемся СРАЗУ (не ждём API)
2. Передаём конфиг в PacketTunnel Extension (отдельный процесс iOS)
3. Extension запускает sing-box с этим конфигом
4. sing-box создаёт TUN интерфейс (виртуальную сетевую карту)
5. Весь трафик iPhone идёт через этот интерфейс → через VPN
6. В фоне: обновляем конфиг с сервера для следующего раза
```

### Как iPhone попадает на сервер API (из России)

Домен razblokirator.ru может быть заблокирован. Поэтому 3 попытки:

```
Попытка 1: https://razblokirator.ru/api/...     (через Cloudflare CDN)
    ↓ не получилось
Попытка 2: http://185.218.0.43/api/...           (SPB relay, Россия)
    ↓ не получилось
Попытка 3: http://162.19.242.30/api/...           (прямой IP в Германии)
```

## Архитектура — кто что делает

### На iPhone
```
┌─────────────────────────────────┐
│  Chameleon App (SwiftUI)        │  ← Красивые кнопочки
│  ├── AppState          — мозг   │
│  ├── APIClient         — скачивает конфиг
│  ├── ConfigStore       — хранит конфиг
│  ├── ConfigSanitizer   — чистит конфиг от старых полей
│  └── VPNManager        — управляет VPN тоннелем
├─────────────────────────────────┤
│  PacketTunnel Extension         │  ← Отдельный процесс iOS
│  └── sing-box (libbox)          │  ← Движок VPN, создаёт тоннель
└─────────────────────────────────┘
```

Важно: App и Extension — это **два разных процесса**. Они общаются через:
- **App Group** — общая папка для файлов (конфиг, логи)
- **gRPC** — командный сокет (переключить сервер, получить пинг)

### На сервере (DE — Германия)

```
┌──────────────────────────────────┐
│  Docker Compose                  │
│  ├── chameleon-backend (Rust)    │  ← API + генерация конфигов
│  ├── xray-core                   │  ← Принимает VLESS подключения
│  ├── hysteria2                   │  ← Принимает Hysteria2 (UDP)
│  ├── nginx                       │  ← Раздаёт API через HTTP
│  ├── PostgreSQL                  │  ← База данных (пользователи)
│  └── Redis                       │  ← Кеш
└──────────────────────────────────┘
```

### На сервере (SPB — Россия)

```
┌──────────────────────────────────┐
│  nginx (stream)                  │  ← Пересылает TCP трафик
│  ├── порт 2096 → DE:2096        │     (VLESS в Германию)
│  └── порт 2098 → NL:2096        │     (VLESS в Нидерланды)
└──────────────────────────────────┘
```
SPB — просто "труба". Не знает что внутри. Просто берёт байты с порта 2096 и отправляет на DE.

## Генерация конфига — как сервер собирает JSON

```
Backend получает запрос: "дай конфиг для user_abc123"

1. Загружает из БД: UUID, short_id пользователя
2. Загружает из .env: список серверов (VPN_SERVERS), ключи Reality, пароль Hysteria2
3. Перебирает ВСЕ включённые протоколы:
   - vless_reality → 4 outbound'а (relay-de, relay-nl, de, nl)
   - hysteria2 → 2 outbound'а (de, nl — relay пропущены, UDP не пройдёт)
   - vless_cdn → пропущен (нет CDN_DOMAIN)
   - warp, anytls, naiveproxy → пропущены (нет паролей)
4. Оборачивает в selector (ручной выбор) + urltest (автовыбор)
5. Добавляет DNS (FakeIP), TUN inbound, маршруты
6. Отдаёт JSON клиенту
```

## Текущие ограничения

1. **DE заблокирован напрямую** — прямое подключение к 162.19.242.30 не работает из России. Только через relay (SPB)
2. **Hysteria2 через relay невозможен** — SPB nginx пересылает только TCP, а Hysteria2 это UDP
3. **CDN Fallback не активен** — нужен домен на Cloudflare (CDN_DOMAIN не задан)
4. **libbox версии 1.11** — не самая новая, приходится избегать deprecated полей
5. **Один relay сервер (SPB)** — если заблокируют его IP, relay перестанет работать

## Версии sing-box и что от чего зависит

```
Наш libbox: sing-box ~1.11.x (вкомпилирован в iOS приложение)

Что deprecated в 1.11 (ругается, но работает):
  ❌ dns outbound → убрали
  ❌ sniff на inbound → убрали (есть в route rules)
  ❌ sniff_override_destination → убрали  
  ❌ dns.strategy → убрали (перенесли на серверы)

Что появится в 1.13 (когда обновим libbox):
  ✅ hijack_dns route action (замена dns outbound)
  ✅ Новый формат inbound fields
```
